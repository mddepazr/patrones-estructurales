Patrones Estructurales

1. Adapter

adapter.png

Explicación del Código

¿Qué hace? Tenemos un sistema europeo que espera un enchufe de clavija fina (`EnchufeEuropeo`), pero solo tenemos uno americano (`EnchufeAmericano`). El `AdaptadorAmericanoAEuropeo` implementa la interfaz europea pero internamente llama al método del enchufe americano.

¿Cómo aplica el patrón? Convierte la interfaz de una clase existente (el enchufe americano) en la interfaz que el cliente espera (la europea), logrando que ambos trabajen juntos sin modificar el código original del enchufe americano. 

// Interfaz esperada
interface EnchufeEuropeo { void conectarFino(); }

// Clase incompatible existente
class EnchufeAmericano { 
    void conectarGrueso() { System.out.println("Conectado con clavijas planas"); } 
}

// El Adaptador
class AdaptadorAmericanoAEuropeo implements EnchufeEuropeo {
    EnchufeAmericano enchufe = new EnchufeAmericano();
    
    public void conectarFino() {
        enchufe.conectarGrueso(); // Traduce la llamada
    }
}

2. Bridge

bridge.png

Explicación del Código

¿Qué hace? Separa el control remoto (la abstracción) del dispositivo electrónico, como una TV (la implementación). El control remoto tiene una referencia al dispositivo, actuando como un puente.

¿Cómo aplica el patrón? Desacopla la interfaz del control remoto de la implementación de la TV. Si mañana inventamos un "Control Remoto Avanzado" o agregamos una "Radio", podemos combinarlos sin tener que crear clases como `ControlBasicoParaTV` o `ControlAvanzadoParaRadio`. Evolucionan por separado. 

// Implementación
interface Dispositivo { void encender(); }
class TV implements Dispositivo { 
    public void encender() { System.out.println("TV Encendida"); } 
}

// Abstracción (El Puente)
abstract class ControlRemoto {
    protected Dispositivo dispositivo; 
    ControlRemoto(Dispositivo d) { this.dispositivo = d; }
    abstract void presionarBoton();
}

class ControlBasico extends ControlRemoto {
    ControlBasico(Dispositivo d) { super(d); }
    void presionarBoton() { dispositivo.encender(); }
}

3. Composite

composite.png

Explicación del Código

¿Qué hace? Modela un sistema de archivos. Un `Archivo` es un elemento individual y una `Carpeta` es un contenedor que puede tener archivos u otras carpetas. Ambos implementan `ComponenteSistema`.

¿Cómo aplica el patrón? Permite que el sistema trate un objeto individual (archivo) y un grupo de objetos (carpeta) de manera idéntica. Al llamar a `mostrarDatos()` en una carpeta, esta llama recursivamente al mismo método en todo su contenido, resolviendo la complejidad jerárquica de forma elegante. 

interface ComponenteSistema { void mostrarDatos(); }

class Archivo implements ComponenteSistema {
    public void mostrarDatos() { System.out.println("  - Archivo.txt"); }
}

class Carpeta implements ComponenteSistema {
    List<ComponenteSistema> hijos = new ArrayList<>();
    void agregar(ComponenteSistema c) { hijos.add(c); }
    
    public void mostrarDatos() {
        System.out.println("[Carpeta Principal]");
        for(ComponenteSistema hijo : hijos) { 
            hijo.mostrarDatos(); // Llamada recursiva uniforme
        }
    }
}

4. Decorator

decorator.png

Explicación del Código

¿Qué hace? Partimos de un `CafeBasico`. El `DecoradorLeche` "envuelve" a este café. Cuando llamamos a `preparar()`, el decorador primero ejecuta el método del café base y luego le añade su propia característica (" + Leche").

¿Cómo aplica el patrón? Evita crear cientos de subclases (como `CafeConLecheYAzucar`). En su lugar, añadimos responsabilidades al objeto en tiempo de ejecución de manera dinámica y escalable. Envolvemos la base tantas veces como queramos con diferentes ingredientes. 

interface Cafe { String preparar(); }

class CafeBasico implements Cafe {
    public String preparar() { return "Café base"; }
}

abstract class CafeDecorador implements Cafe {
    protected Cafe cafeBase;
    CafeDecorador(Cafe c) { this.cafeBase = c; }
}

class DecoradorLeche extends CafeDecorador {
    DecoradorLeche(Cafe c) { super(c); }
    public String preparar() { 
        // Llama a la base y agrega su comportamiento
        return cafeBase.preparar() + " + Leche"; 
    }
}

5. Facade

facade.png

Explicación del Código

¿Qué hace? Tenemos subsistemas complejos (`Luces`, `Proyector`). El cliente no interactúa directamente con ellos. En su lugar, llama al método `modoPelicula()` de la clase `CineEnCasaFacade`, que se encarga de coordinar el apagado de luces y el encendido del proyector.

¿Cómo aplica el patrón? Proporciona una interfaz unificada, simple y de alto nivel. Oculta la complejidad de los subsistemas, haciendo que el sistema principal sea mucho más fácil de utilizar y entender para el cliente final. 

// Subsistemas complejos
class Luces { void apagar() { System.out.println("Luces apagadas"); } }
class Proyector { void encender() { System.out.println("Proyector encendido"); } }

// La Fachada (Facade)
class CineEnCasaFacade {
    Luces luces = new Luces();
    Proyector proyector = new Proyector();
    
    // Método simplificado para el cliente
    void modoPelicula() {
        luces.apagar();
        proyector.encender();
    }
}
